PROFESSOR FEEDBACK THAT MUST BE ADDRESSED

The professor raised the following UX questions that must be resolved through improvements:

1. **Teacher Flow Efficiency**

Can a teacher complete their tasks in a few seconds?

The system must allow extremely fast actions during real supervision.

Check especially:

• starting a shift
• check-in with QR/PIN
• confirming patrol movement
• reporting an incident

All these actions should require **maximum 3 taps**.

If the current interface requires more steps, optimize it.

2. **Visual Accessibility**

Teachers may use the system:

• while walking
• outdoors under sunlight
• with only one hand

Improve if necessary:

• button size
• contrast
• text readability
• spacing between elements
• touch targets

Buttons for main actions must be clearly visible.

3. **Notifications**

Notifications must clearly show:

Zone
Time
Quick action button

Example:

"Recess duty in 5 minutes
Zone: North Playground
[Open Shift]"

If the current notifications are not clear enough, improve their structure.

4. **Coordinator Dashboard**

The coordinator interface is critical.

The coordinator must be able to quickly:

• detect uncovered zones
• see active supervision
• reassign shifts quickly

Check if the board feels cluttered.

Improve:

• visual hierarchy
• color clarity (green, yellow, red states)
• speed of reassignment actions
